Casos de Estudio y Ejemplos de Aplicación

Caso de Estudio 1: Generador de Documentos

Una empresa necesita generar documentos en varios formatos (PDF, HTML, Word) a partir de la misma fuente de datos.

Patron Aplicado: Factory Method

Problema Queremos crear objetos de diferentes tipos de documentos (PDF, HTML, Word) sin especificar sus clases concretas en el código cliente.
Solucion Utilizamos el patrón Factory Method para definir una interfaz de creación de objetos, dejando a las subclases la decisión de qué tipo de objeto crear.

Implementación en Java

Factory.java

Caso de Estudio 2: Sistema de Gestión de Usuarios

Un sistema necesita manejar diferentes tipos de usuarios (administradores, clientes, proveedores) con comportamientos específicos.

Patron Aplicado: Strategy

Problema Queremos definir una familia de algoritmos (comportamientos de usuario) y hacerlos intercambiables en tiempo de ejecución.
Solucion Utilizamos el patrón Strategy para encapsular los comportamientos de usuario en clases separadas y permitir su selección dinámica.

Implementación en Java
